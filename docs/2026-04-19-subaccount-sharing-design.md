# 子账号选择性共享功能设计

## 概述

管理员可为子账号配置细粒度的数据共享权限，支持选择具体的日志、词典、历史记录分享给指定的子账号。

## 数据模型

### shares 表改动

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键 |
| from_user_id | VARCHAR(36) | 分享方用户ID |
| to_user_id | VARCHAR(36) | 接收方用户ID |
| share_type | ENUM | 'logs', 'dictionaries', 'history', 'both' |
| item_ids | JSON | 具体共享项目ID列表，null表示全部 |
| auto_sync | BOOLEAN | 新增是否自动共享，默认false |
| status | ENUM | 'active', 'inactive' |
| created_at | DATETIME | 创建时间 |

## API 设计

### 管理员接口

#### GET /admin/shares
获取所有共享配置列表

#### GET /admin/shares/:userId
获取指定用户的共享配置详情（包含已选项目）

#### PUT /admin/shares/:userId
更新用户的共享配置
```json
{
  "shareType": "logs",
  "itemIds": ["id1", "id2"],
  "toUserIds": ["userId1", "userId2"],
  "autoSync": true
}
```

#### DELETE /admin/shares/:userId/:targetUserId
删除指定共享关系

### 共享查询接口

#### GET /sync/shared-logs
获取共享给我的日志（包含分享者信息）

#### GET /sync/shared-dictionaries
获取共享给我的词典

## 共享流程

1. 管理员选择源子账号，点击配置共享
2. 选择共享类型（日志/词典/历史）
3. 勾选要共享的具体项目，或全选
4. 开启"全选新增自动共享"则后续新增自动同步
5. 选择目标子账号（可多选）
6. 保存后建立单向只读共享关系

## 权限规则

- 只有管理员可配置共享
- 子账号无法主动发起共享（只能管理员代为配置）
- 共享为单向只读，接收方无法修改或再次分享
- auto_sync 仅对新增项目生效，已有项目不受影响
