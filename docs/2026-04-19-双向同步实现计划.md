# 全量双向同步 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 OpenLogTool 客户端与 OpenLogToolServer 实现覆盖日志、词典、呼号-QTH 历史、history 归档的真正双向同步，并统一支持软删除、较新更新时间优先冲突解决，以及 realtime / interval / manual 三种触发模式。

**Architecture:** 先统一客户端与服务端的数据格式和同步元数据，再为四类资源补齐 `findSince / upsert / softDelete` 能力，最后把服务端 `/logs/sync/bidirectional` 和客户端 `SyncProvider` 收敛到同一个双向同步入口。自动同步只负责触发统一入口，不再分裂为 UI 私有逻辑和 provider 私有逻辑两套实现。

**Tech Stack:** Flutter + Provider + SQLite(sqflite), Node.js + Express, MySQL / MongoDB / Memory adapters.
